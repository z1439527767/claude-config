# 思考流程（PIRATE + 双模）
> 🔴 Must | 每次任务，先想清楚再动手
> 融合：Junie PIRATE 框架 + Devin 双模架构 + Codex 前缀规范

## PIRATE 六步
1. **Plan** — 复杂任务先规划，简单任务直接做。新项目有创意，已有代码手术刀般精确。
2. **Iterate** — 小步迭代。一次一个任务，先最简原型再叠加。
3. **Review** — 改完逐行审查。所有反馈合并到一条消息（省 token）。
4. **Assess** — 真能跑吗？有副作用吗？不只是看代码，要看效果。
5. **Test** — 每个改动验证。先用最近测试、再扩大范围。AI 生成的测试也要审查。
6. **Evaluate** — 反思这次做对了什么、做错了什么。学到的及时写配置。

## 双模切换
- **探索模式**（Planning）：只读，查代码库，产出计划。不改文件。
- **执行模式**（Standard）：写代码，跑测试，验证闭环。

## 动手前自问（回忆→目标→范围→路线→根因→风险）
**回忆** — 当前项目上次学到了什么？查 CLAUDE.local.md + 知識圖譜。
**目标** — 用户要什么结果？一句说清。
**范围** — 涉及哪些文件？最简单改哪里？
**路线** — 先改什么、再改什么、怎么验证？能并行的不串行。
**根因** — 症状还是根因？同问题两次 = 停，找根因。
**质疑** — 这是最简单的方法吗？反过来想，不写代码能不能达到同样效果？

## 做完后闭环（验证→收尾→优化→沉淀）
**验证** — 外部手段确认改动生效。自审查盲点 64.5%——考虑跨模型 review。
**收尾** — 有没有遗留？关联文件都改了吗？
**优化** — 到极限了吗？还能更简单吗？
**沉淀** — 值得记住的写 CLAUDE.local.md + 知識圖譜。同错两次写规则。同任务三次写 skill。
