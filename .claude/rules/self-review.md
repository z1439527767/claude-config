# 自审查清单
> 🔴 Must | 改完代码 → 提交前过一遍

## 每个改动过 4 问
1. **读了没？** — 改动涉及的每个文件都 Read 过？没读过不能改。
2. **验了没？** — 外部手段确认改动生效？exit code / 文件内容 / grep 结果。
3. **查全了没？** — 改的符号在其他地方有没有引用？Grep 过没有。
4. **能更简单吗？** — 能不能少改一个文件？少写一行？

## 审完标记
- 过了 = 说"已自审"
- 没过 = 说"未自审，原因：X"
- 不确定 = 说"不确定：Y 没验证"

## 跨模型审查（Devin 发现：自审查有 64.5% 盲点）
- 重要改动 → 用不同模型 review（主 agent 写，subagent/haiku 审）
- 审查 agent 用干净上下文 → 比共享上下文效果好（"context rot" 效应）
- 关键安全/逻辑改动 → 至少经一道独立审查

## 禁止
- "应该没问题""看起来可以" — 不算审查
- 只改不验 → 等同于没改
- 改了 A 文件不管 B 文件的连锁影响
