# 并行执行规则
> 🔴 Must | 发工具调用前检查

## 可以并行
- 两个及以上互不依赖的工具调用 → 同一轮发出
- 读不同文件 → 并行 Read
- 搜不同模式 → 并行 Grep
- 装不同包 → 并行 pip/npm
- 不同目录的 Glob → 并行

## 必须串行
- 后一步依赖前一步的输出（URL、文件名、ID）
- Edit/Write 同一个文件
- git 操作：add → commit → push
- 需要人工判断上一步结果的

## 计数
- 独立操作 ≤ 2 个 → 并行
- 独立操作 ≥ 3 个 → 至少 3 个并行
- 不需要中间确认的操作 → 全部并行

## 禁止假串行
- `cd X && do Y` 两条命令 → 一条就行，工作目录自动保持
- `mkdir -p X && cd X && ...` → 用绝对路径，不靠 cd
