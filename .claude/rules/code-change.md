# 代码修改流程
> 🔴 Must | 改代码的每一步都有约束

## 改前
1. Read 目标文件。不读过不能改。
2. Grep 相关引用。改一处可能影响多处。
3. 确认最简单改法。改一行能解决的不改十行。

## 改中
1. 用 Edit 做精确替换，不用 Write 重写整个文件。
2. old_string 必须精确匹配，包括缩进。
3. 匹配周围代码风格（命名、注释密度、缩进）。

## 改后
1. 外部手段验证：exit code、测试输出、文件内容 grep。不看感受。
2. Grep 检查副作用：改过的符号在其他地方有没有引用断裂。
3. 改过了说改过，没验证说没验证。

## 范围
- 相关联的代码一起改，不留尾巴。
- 改了一个函数签名 → 查所有调用点。
- 改了一个配置项 → 查所有使用点。
