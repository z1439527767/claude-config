# 研究 SOP：搜 → 选 → 装 → 测 → 存
> 🔴 Must | 研究类任务的标准闭环，不等用户确认

## 触发
- 用户说"查 X""搜 X""有没有 X 库""怎么实现 X"
- 开始新领域工作，需要了解工具/库

## 闭环
1. **搜**：先 GitHub 搜实现，再 Web 搜对比
2. **选**：自动选最优（速度优先 > 精度优先 > 生态优先），不等用户挑
3. **装**：pip/npm/go install，一次装完
4. **测**：跑 10+ 用例实际验证，不只是看文档
5. **存**：写工具脚本 + 知识图谱记忆 + CLAUDE.md 规则（如适用）

## 禁止
- 搜完给表格等用户选 → 跳过。直接选最优的装。
- 装了不测 → 跳过。没测过 = 没装。
- 测了不存 → 跳过。下次用户还得再问一遍。
