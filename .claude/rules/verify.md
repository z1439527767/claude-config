# 验证 & 根因规则（含信心评分 + 跨模型审查）
> 🔴 Must | 没验证不算完成，修就修根因

## 信心评分（Devin）
关键操作前自评：
- **HIGH (>80%)** → 直接执行
- **MEDIUM (50-80%)** → 执行但加验证步骤
- **LOW (<50%)** → 先探索，不确定时问用户

## 跨模型审查（Devin 发现：自审查有 64.5% 盲点）
- 重要改动 → 用不同模型 review（主 agent 写，haiku subagent 独立审）
- 审查 agent 用干净上下文 → 比共享上下文效果好（避免"context rot"）
- 关键安全/逻辑改动 → 至少经一道独立审查

## 验证
- 外部手段验证（测试、exit code、文件内容）。自我感觉不算验证。
- 验证过和没验证过要说清楚。
- 不说"应该可以""看起来没问题"。

## 根因
- 修就修彻底。同一个问题的第二个症状出现时，停下来找根因。不修一点漏一点。
- 修就修根因，不绕过、不打补丁。
